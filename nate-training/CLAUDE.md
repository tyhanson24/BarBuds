# Nate Training App

Mobile-first 12-week post-ACL summer football training tracker.
Single user (Nate, 20yo D3 safety at Ripon, 7 months post-op LEFT ACL/meniscectomy).
Read-write app on his phone in the garage.

## Stack
- Vite + React 18 + TypeScript (strict)
- Tailwind 3, lucide-react, date-fns
- @supabase/supabase-js (PrepScores project, tables prefixed `nate_`)
- Deploy: Vercel, root dir = `nate-training`

## Conventions
- All workout data generated by templates in `src/data/workouts.ts` — DO NOT inline new workouts elsewhere
- All times/dates use date-fns; program day math derives from `PROGRAM_START_DATE` in `src/data/program.ts`
- Supabase tables: `nate_settings`, `nate_workout_completions`, `nate_exercise_logs`, `nate_body_metrics`, `nate_test_results`
- Storage API lives in `src/lib/storage.ts` — no other file should call `supabase` directly for app data
- No router. Single-page tab state via `useState`
- No external state library
- Strict TypeScript — every Supabase row maps to a typed shape in `src/types.ts`

## Visual non-negotiables
- Dark UI only (ink-900 background)
- Single accent: Ripon red (#c8102e)
- Bebas Neue for display, JetBrains Mono for all numbers, Archivo for body
- Phase color as a 4px left border on cards

## Single-user auth model
- No login. `VITE_NATE_USER_ID` in `.env.local` is hardcoded.
- All RLS policies on `nate_*` tables check `user_id = <NATE_UUID>` against anon role.
- Don't generalize this to multi-user without first adding real auth.

## Commands
- `npm run dev` — local dev (port 5173)
- `npm run build` — production build (tsc -b + vite build)
- `npm run preview` — preview prod build
